在数据分析中，回归分析是常用的统计方法，尤其在探索变量之间的关系时。在本教程中，我们将通过一个实际的例子，演示如何使用R语言高效地进行回归分析，批量处理不同类型的暴露变量（包括分类变量和连续变量），并生成回归结果表格。我们将通过自定义函数来批量处理不同模型和变量类型，生成清晰的回归分析结果，帮助你轻松得出结论。

01

数据准备与回归配置

在进行回归分析之前，首先需要加载数据，并配置不同的回归模型。数据中的暴露变量包括分类变量、连续变量和有序变量，目标是研究这些暴露变量与抑郁评分（phq9_score）之间的关系。


在这部分代码中，我们首先读取清理后的数据集，并查看数据结构。接着定义了三种不同的回归模型，每个模型配置了不同的协变量。

02

函数1：处理分类暴露变量的回归

对于分类暴露变量（例如：screen_time_group），我们使用run_categorical_regression函数进行回归分析。该函数会自动根据暴露变量的分类水平（如：低、中、高）计算回归系数，并输出结果。



该函数的关键步骤是通过glm函数拟合回归模型，并自动输出回归结果（包括OR或Beta值及95%置信区间）。如果有趋势检验，还会计算对应的p值。

03

函数2：处理连续暴露变量的回归

对于连续暴露变量（例如：daily_screen_hours），我们使用run_continuous_regression函数进行回归分析，输出回归系数及其95%置信区间。


此函数通过glm拟合模型，并输出回归系数（OR或Beta值），如果是逻辑回归，还会返回相应的置信区间。

04

批量回归分析流程

接下来，我们定义了暴露变量列表，并对每种暴露变量进行批量回归分析。每种暴露变量会分别应用于不同的模型，并生成回归分析结果表格。

05

函数1 使用说明

📌 函数 1：run_categorical_regression()

项目 说明
用途 对分类暴露变量（如 screen_time_group）进行线性回归（因结局为连续变量 phq9_score），并同时计算趋势检验（使用有序变量如 screen_time_quartile）。
适用场景 暴露变量为因子型（factor），有多个水平（如“低/中/高”），需以第一个水平为参照，估计其他水平的效应，并可选进行线性趋势检验。
输入参数
data 数据框（data.frame），包含所有变量。
outcome_var 字符串，结局变量名（如 "phq9_score"）。
exposure_cat 字符串，分类暴露变量名（如 "screen_time_group"）。必须是因子（factor）类型。
exposure_trend 字符串，用于趋势检验的有序变量名（如 "screen_time_quartile"），若不进行趋势检验可设为 NA。
covariates 字符向量，协变量列表（如 c("age", "sex")），若为空则只调整截距（即 crude model）。
model_name 字符串，当前模型名称（如 "model2"），用于结果标识。
输出内容 返回一个数据框，包含：
• Exposure：暴露变量名
• Model：模型名称
• Q1, Q2, ...：各组相对于参照组的效应估计（格式为 "Est (Lower, Upper)"）
• p for trend：趋势检验的 p 值（若提供 exposure_trend）
注意事项

  • 要求 exposure_cat 是因子且已设置参照组（默认第一水平为参照）。

  • 若 exposure_trend 为 NA，则 p for trend 列为 NA。

  • 因 phq9_score 是连续变量，自动使用 gaussian 族（即线性回归）。

06

函数2 使用说明

📌 函数 2：run_continuous_regression()

项目 说明
用途 对连续型暴露变量（如 daily_screen_hours 或 screen_time_zscore）进行回归分析，估计每单位变化对结局的影响。
适用场景 暴露变量为数值型（numeric），需报告回归系数（Beta）及其 95% 置信区间和 p 值。
输入参数
data 数据框（data.frame）。
outcome_var 字符串，结局变量名（如 "phq9_score"）。
exposure_cont 字符串，连续暴露变量名（如 "daily_screen_hours"）。
covariates 字符向量，协变量列表（可为空）。
model_name 字符串，模型名称（如 "model3"）。
输出内容 返回一个数据框，包含：
• Exposure：暴露变量名
• Model：模型名称
• Effect (95% CI)：格式为 "Beta (Lower, Upper)"
• P value：对应 p 值（保留三位小数）
注意事项

  • 自动检测结局是否为因子以决定使用 logistic 还是 linear 回归（本例中为 linear）。
